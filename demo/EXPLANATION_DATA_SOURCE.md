# 시연 데이터 출처 및 신뢰성 설명

## 📊 데이터 출처

### 현재 시연용 샘플 데이터 (`sample_user_profile.json`)

**상태**: ⚠️ **시뮬레이션 데이터** (실제 사용자 데이터 아님)

**생성 방법**:
- 논문의 알고리즘과 수치를 기반으로 생성
- 논리적 일관성을 유지하도록 설계
- 실제 시스템 동작 방식을 반영

**데이터 특성**:
- 세션 1→7로 진행하면서 신뢰도 증가 (30% → 82%)
- Logic 가중치 증가, Intuition 감소 (분석적 성향 강화)
- 경로 효율성 개선, 의사결정 지연시간 증가
- 논문의 "지속적 학습" 패턴을 모방

### 실제 데이터베이스 (`user_profiles.db`)

**상태**: ✅ **실제 시스템 데이터베이스**

**구조**:
- SQLite 데이터베이스
- 실제 사용자 세션 데이터 저장 가능
- GDPR 준수 (동의 기록, 데이터 삭제 기능)

**현재 상태**:
- 데이터베이스 스키마는 완성됨
- 실제 사용자 세션이 있으면 저장됨
- 시연용으로는 아직 실제 데이터가 적을 수 있음

## 🤔 왜 샘플 데이터를 사용했나?

### 이유 1: 실제 데이터 부족
- 프로토타입 단계에서는 실제 사용자 데이터가 제한적
- 여러 세션(1, 3, 5, 7)의 연속된 데이터가 필요
- 발표 시 일관된 시연을 위해

### 이유 2: 논리적 일관성
- 샘플 데이터는 논문의 알고리즘을 정확히 반영
- 실제 시스템이 생성할 수 있는 결과와 일치
- 발표 시 설명하기 쉬운 패턴

### 이유 3: 재현 가능성
- 동일한 데이터로 반복 시연 가능
- 예상치 못한 변수 없이 안정적인 발표
- 실제 시스템 오류 시 백업으로 사용

## ✅ 실제 데이터로 교체하는 방법

### 방법 1: 실제 시스템에서 데이터 수집

```bash
# 1. 백엔드 실행
cd backend
python api_server.py

# 2. 프론트엔드 실행
npm run dev

# 3. 실제로 여러 세션 진행
# - IdentityConfigurator에서 선택
# - 각 세션마다 /api/session 호출
# - 데이터베이스에 저장됨
```

### 방법 2: 데이터베이스에서 추출

```python
from backend.user_profiles import UserProfileManager

manager = UserProfileManager()
user_id = "실제_사용자_ID"

# 사용자 데이터 내보내기
data = manager.export_user_data(user_id)

# JSON 파일로 저장
import json
with open('demo/real_user_profile.json', 'w', encoding='utf-8') as f:
    json.dump(data, f, ensure_ascii=False, indent=2)
```

### 방법 3: 실제 데이터로 차트 재생성

```bash
cd demo
# sample_user_profile.json을 real_user_profile.json으로 교체
python generate_demo_charts.py
```

## 📈 데이터 신뢰성

### 알고리즘 기반 생성
- ✅ 논문의 수식과 로직을 정확히 반영
- ✅ 실제 시스템 코드와 일치
- ✅ 논리적 일관성 유지

### 실제 시스템과의 차이
- ⚠️ 실제 사용자는 더 다양한 패턴을 보일 수 있음
- ⚠️ 노이즈와 변동성이 있을 수 있음
- ⚠️ 이상치(outlier)가 발생할 수 있음

### 검증 방법
1. **코드 검증**: 샘플 데이터가 실제 알고리즘으로 생성 가능한지 확인
2. **논리 검증**: 수치가 논리적으로 일관성 있는지 확인
3. **실제 테스트**: 실제 사용자로 테스트하여 패턴 비교
